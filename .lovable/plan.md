

## Google Authentication

Lovable Cloud has **managed Google OAuth** built in — no API keys or configuration needed.

### What needs to happen

1. **Generate the Lovable Cloud auth module** using the Configure Social Login tool — this creates the `src/integrations/lovable/` folder with the Google OAuth helper
2. **Add a "Sign in with Google" button** to both `Login.tsx` and `Register.tsx` using `lovable.auth.signInWithOAuth("google")`
3. **Update `Index.tsx`** to handle the OAuth redirect callback (user lands back at origin after Google sign-in)

### Does this impact the data model?

**No.** Google OAuth uses the same underlying auth system. The user ends up in `auth.users` just like email/password signups. When we later create the `profiles` table, we'll use a trigger that fires on any new auth user — regardless of sign-in method. The `display_name` and `avatar_url` can be pulled from Google's user metadata automatically.
